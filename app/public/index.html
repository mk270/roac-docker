Basic catalogue

This webpage demostrates how we can build a shared catalogue: we agree a rough consensus on what data we should hold on books. This is then exposed read-only via GraphQL, a standard for making APIs for data. The system has been developed broadly in accordance with Indieweb principles and uses open source software and open standards.

Intro

The data model lets us record data about the following sorts of things:

  • books
  • chapters of books
  • publications (e.g., the paperback version of a book)
  • contributors, including authors, illustrators, editors
  • publishers
  • who contributed to which book
  • ... and a few other matters

OBP and Punctum have been careful to specify which data is absolutely mandatory and which data is optional. That is to say, the system will summarily reject books that don't have titles, but will not require that they have a DOI.
